# Morning Coffee

A daily focus workflow for [Claude Code](https://docs.anthropic.com/en/docs/claude-code). Plan your day, stay on track, and close the loop — all driven by a single workboard file.

## The Workflow

```
Morning                      During the day              End of day
┌──────────────┐            ┌──────────────┐            ┌──────────────┐
│ /morning-    │            │              │            │              │
│  coffee      │───────────▶│   /today     │───────────▶│  /wrap-up    │
│              │            │              │            │              │
│ Read board   │            │ Quick ref    │            │ What got done│
│ Pull context │            │ for today's  │            │ Carry-forward│
│ Pick focuses │            │ plan         │            │ Update board │
└──────────────┘            └──────────────┘            └──────────────┘
```

## What's Included

| Component | Description |
|-----------|-------------|
| `/morning-coffee` | Daily focus picker — reads workboard, pulls calendar/email/triage, helps pick 2-3 focuses |
| `/today` | Quick reference for today's plan |
| `/wrap-up` | End-of-day review — compare plan vs actual, update workboard |
| `/workboard-init` | Optional guided workboard bootstrapping |
| Provider system | Pluggable calendar/email integration (duo-msgraph default) |
| Triage integration | Interface for pre-computed message triage |

## Prerequisites

- [Claude Code](https://docs.anthropic.com/en/docs/claude-code) installed and working
- That's it. Calendar, email, and triage are optional enhancements.

## Quick Start

```bash
git clone https://github.com/your-org/morning-coffee.git
cd morning-coffee
./setup.sh
```

The setup script will:
1. Install skills to `~/.claude/commands/`
2. Create the memory directory structure
3. Walk you through provider configuration (optional)
4. Set up your workboard (template, example, or guided)

See [docs/quickstart.md](docs/quickstart.md) for the full walkthrough.

## How It Works

**The workboard** (`workboard.md`) is the backbone — a single markdown file tracking all your active, parked, and dropped work. Each item has a state, next step, blockers, and who you're waiting on.

**Each morning**, `/morning-coffee` reads your workboard, optionally pulls calendar/email/triage context, and helps you pick 2-3 focus items. It flags stalled work, surfaces reminders, and writes a daily plan file.

**During the day**, `/today` gives you a quick glance at what you committed to.

**At end of day**, `/wrap-up` compares what you planned vs what happened, captures carry-forwards, and updates the workboard for tomorrow.

## Provider System

Morning-coffee uses a provider system for calendar and email. If no provider is configured, it works fine with just the workboard.

| Provider | Calendar | Email | Status |
|----------|----------|-------|--------|
| [duo-msgraph](providers/duo-msgraph/) | Microsoft Outlook | Microsoft Outlook | Default for Cisco/Duo |
| [google-workspace](providers/google-workspace/) | Google Calendar | Gmail | Community stub |
| [none](providers/none/) | — | — | Workboard-only mode |

See [providers/README.md](providers/README.md) for how to write your own.

## Message Triage

Morning-coffee can consume a pre-computed triage file that categorizes your messaging into: Blocked on You, Decisions Made Without You, Opportunities, and FYI.

This is generated by a separate agent (reference implementation: Webex triage agent). See [integrations/webex-triage.md](integrations/webex-triage.md) for the interface contract and setup guide.
